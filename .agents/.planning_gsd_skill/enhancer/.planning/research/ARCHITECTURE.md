# Architecture Patterns: Darktable GenAI Enhancer

**Domain:** Photography Automation
**Researched:** 2025-05-01

## Recommended Architecture: Orchestrated Sidecar Injection

The system follows a "Sidecar-First" pattern, where the AI never touches the original RAW file or the Darktable database directly. Instead, it generates and modifies XMP sidecar files.

### Component Boundaries

| Component | Responsibility | Communicates With |
|-----------|---------------|-------------------|
| **Metadata Orchestrator** | Extracts EXIF/Lens data. | ExifTool |
| **Analysis Engine** | Vision analysis + Subject Research. | LangChain / LLM |
| **XMP Injector** | Packs binary params and writes sidecars. | lxml / struct |
| **Render Engine** | Exports previews or final JPGs. | darktable-cli |
| **Mentor Interface** | Formats educational "Pre-flight" reports. | Rich (Terminal) |

### Data Flow

1.  **Ingest**: Path to RAW file is provided.
2.  **Metadata**: `ExifTool` extracts Camera/Lens/Settings.
3.  **Research**: AI detects subject (e.g., "Bird") and queries `ChromaDB` for best AgX practices for that subject.
4.  **Generation**: AI decides on module settings (Exposure, AgX Purity, Sigmoid Skew).
5.  **Injection**: Python `struct` packs these into hex-binary blobs; `lxml` updates the `.xmp`.
6.  **Rendering**: `darktable-cli` renders a preview with the new XMP.
7.  **Reporting**: `Rich` prints the "Pre-flight Review" to the user.

## Patterns to Follow

### Pattern 1: Surgical XMP Injection
Avoid rewriting the entire XMP. Only target the `<darktable:history>` list to maintain user metadata (tags, ratings).

### Pattern 2: Binary Param Packing
Darktable parameters are C-structs. Use Python's `struct` module to ensure bit-perfect compatibility.
**Example:**
```python
import struct
# Packing a float for AgX 'pivot' setting
# 'f' = float (4 bytes), '<' = little-endian
pivot_binary = struct.pack('<f', 0.18) 
pivot_hex = pivot_binary.hex()
```

## Anti-Patterns to Avoid

### Anti-Pattern 1: Database Locking
**What:** Running `darktable-cli` while the Darktable GUI is open without specific flags.
**Why bad:** Causes "Database is locked" errors and potential data loss.
**Instead:** Always use `--apply-custom-presets false` and `--core --conf resourcelevel=high` for CLI tasks.

### Anti-Pattern 2: Hard-coding Module Versions
**What:** Hard-coding `modversion="7"` for the AgX module.
**Why bad:** Darktable updates module versions frequently.
**Instead:** Read the `modversion` from a "template" XMP generated by the user's specific Darktable version.
